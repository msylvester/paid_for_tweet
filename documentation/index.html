Main Folders Explained

dist/ folder
 
This folder contains the compiled files.
 
master/ folder
 
This folder contains the source files. You will find the following folders inside
 
sass/ This folder contains the SCSS files for the core styles and bootstrap styles.
jsx/ Here you will find JSX files for React components. All this files are concatenated into the file app.bundle.js.
img/ Images used in the copy (only copied to dist folder)
server/ Contains json files used to emulate server calls

Build

Node.js is a platform built on Chrome’s JavaScript runtime for easily building fast, scalable network applications.
 
Gulp is a task manager, you can define different tasks to run certain commands. Those commands does the compilation job for JSX and SASS files
 
Bower is a dependency manager, it works by fetching and installing packages from all over, taking care of hunting, finding, downloading, and saving the stuff you’re looking for. Bower keeps track of these packages in a manifest file, bower.json.
 
The package includes under the master/ folder the file gulpfile.js, package.json and bower.json to install the required components to compile the source files.
 

>Installing tools

 
 
  • To install node and npm, go to http://nodejs.org/
  • Run npm install -g bower to install bower to manage dependencies
  • Download and install GIT for your platform http://git-scm.com/downloads
 
Once you have all tools installed
 
  • Open a terminal, in the root folder, run the command npm install. This command will install gulp and all project dependencies.
  • Then, to install vendor dependencies, run bower install
  • Finally run gulp to start the task manager (see below for more information)
If everything goes fine, you should see the messages in the terminal telling you that most the task are done ok. The task will watch for files to compile them automatically all files when change
 

 

Main Gulp Tasks

 
  1. $ gulp

 This command will run the default task without minify assets 

2.  $ gulp serve

This taks will start the app with a server in your favourite browser

 

 

3,   $ gulp serve --prod

This taks will start the app with a server in your favourite browser in production mode

 

 

4. $ gulp build --prod

This taks will generate the asset minified (without watch)

Webpack and React Hot Loader

Internally, the application is built using WebPack which is a tool that analyze the code and builds a script ready for web browsers.

The configuration of webpack is included in files webpack.config.dev.js and webpack.config.prod.js for each environment respectively.

For development stage, gulp will enable a webpack plugin called react-hot-loader which allows to reload components easily without waiting to recompile the entire app.
